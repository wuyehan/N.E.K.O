# Подключение NEKO к QwenPaw

## Руководство по установке QwenPaw

### Шаг 1. Установка

Python не нужно настраивать вручную. Одной командой автоматически устанавливаются `uv`, виртуальное окружение, QwenPaw и его зависимости. Обратите внимание: в некоторых сетевых средах или при корпоративных ограничениях прав это может не сработать.

macOS / Linux:

```bash
curl -fsSL https://qwenpaw.agentscope.io/install.sh | bash
```

Windows (PowerShell):

```powershell
irm https://qwenpaw.agentscope.io/install.ps1 | iex
```

### Шаг 2. Инициализация

После установки откройте новый терминал и выполните:

```bash
qwenpaw init --defaults
```

Во время инициализации появится предупреждение о безопасности. QwenPaw объясняет, что работает в вашей локальной среде, и если один экземпляр используют несколько человек, они будут делить доступ к файлам, командам и секретам. Прочитайте предупреждение и введите `yes`, чтобы продолжить.

![Предупреждение безопасности при инициализации QwenPaw](assets/openclaw_guide/image1.png)

### Шаг 3. Запуск

```bash
qwenpaw app
```

Если всё прошло успешно, в конце терминала обычно появится:

```text
INFO:     Uvicorn running on http://127.0.0.1:8088 (Press CTRL+C to quit)
```

После запуска откройте `http://127.0.0.1:8088`, чтобы перейти в консоль QwenPaw.

### Шаг 4. Замена файлов персонажа (необязательно)

После инициализации QwenPaw автоматически создаёт каталог конфигурации:

- Windows по умолчанию: `C:\Users\ИмяПользователя\.qwenpaw`
- macOS по умолчанию: `~/.qwenpaw`

Папка `.qwenpaw` скрыта, поэтому при необходимости включите отображение скрытых файлов.

- Windows: показать скрытые элементы в Проводнике
- macOS: нажать `Command + Shift + .` в Finder

Если вы хотите, чтобы QwenPaw работал как чистый фоновый исполнитель для N.E.K.O, скачайте этот архив:

- [Файлы замены.zip](assets/openclaw_guide/替换文件.zip)

Скопируйте `SOUL.md`, `AGENTS.md` и `PROFILE.md` из архива в `.qwenpaw/workspaces/default` с заменой существующих файлов и удалите `BOOTSTRAP.md` из этой папки.

Затем остановите QwenPaw через `CTRL+C` и запустите снова:

```bash
qwenpaw app
```

## Базовая настройка: модель

Откройте консоль QwenPaw, перейдите на страницу моделей и выберите провайдера, которого хотите использовать. Для начинающих часто подходит `DashScope`, но при необходимости можно выбрать другой вариант под ваш API Key.

Откройте настройки провайдера, введите API Key и сохраните изменения.

![Страница настройки модели QwenPaw](assets/openclaw_guide/image2.png)

После сохранения вернитесь в чат и выберите настроенную модель.

## Включение OpenClaw в N.E.K.O

Внутреннее имя в N.E.K.O по-прежнему `openclaw`, поэтому переключатель `OpenClaw` в интерфейсе фактически означает QwenPaw.

Действуйте в таком порядке:

1. Откройте панель Agent в N.E.K.O
2. Включите главный переключатель `Agent`
3. Убедитесь, что `openclawUrl` указывает на `http://127.0.0.1:8088`
4. Затем включите подпункт `OpenClaw`
5. Дождитесь успешной проверки доступности

N.E.K.O сначала пробует совместимый эндпоинт QwenPaw, а при необходимости автоматически переключается на основной `process` эндпоинт. Для основной цепочки подключения настраивать кастомный канал не требуется.
